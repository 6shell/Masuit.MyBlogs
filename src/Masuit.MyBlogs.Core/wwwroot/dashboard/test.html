计数器 Mixin

const counterMixin = {
  data() {
    return {
      counter: 0
    }
  },
  methods: {
    increment() {
      this.counter++;
    },
    decrement() {
      this.counter--;
    },
    resetCounter() {
      this.counter = 0;
    }
  },
  mounted() {
    console.log('计数器Mixin已加载');
  }
};

用户信息 Mixin

const userMixin = {
  data() {
    return {
      userName: '默认用户',
      userId: Math.floor(Math.random() * 1000),
      registrationDate: new Date().toLocaleDateString()
    }
  },
  methods: {
    updateUserInfo() {
      this.userName = '用户' + Math.floor(Math.random() * 1000);
      this.userId = Math.floor(Math.random() * 1000);
      this.registrationDate = new Date().toLocaleDateString();
    }
  },
  mounted() {
    console.log('用户信息Mixin已加载');
  }
};

什么是Mixin？

Mixin是一种在Vue组件中复用功能的方式。它允许你将一组组件选项（如数据、方法、生命周期钩子等）提取到可重用的对象中。

Mixin的优势：

  • 代码复用：避免在不同组件中重复编写相同的代码
  • 逻辑分离：将相关功能组织在一起，提高代码可维护性
  • 灵活性：可以轻松地在多个组件中应用相同的功能

Mixin的合并策略：

  • 数据对象：会进行浅合并，组件数据优先
  • 方法：同名方法会被合并，组件方法覆盖Mixin方法
  • 生命周期钩子：都会被调用，Mixin钩子先于组件钩子

Mixin使用场景

Mixin特别适合以下场景：

  • 多个组件需要共享相同的方法或计算属性
  • 需要在多个组件中执行相同的生命周期操作
  • 封装通用的UI交互逻辑（如模态框、下拉菜单等）
  • 实现横切关注点（如日志记录、权限检查等）

注意事项：

  • 避免过度使用Mixin，以免造成代码难以理解
  • 注意命名冲突，特别是在大型项目中
  • 考虑使用Composition API作为Mixin的替代方案
